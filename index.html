EBWEBS

Mission

Fournir aux utilisateurs avancés et aux professionnels un outil d'information système fiable, lisible et libre — sans publicité, sans tracking, sans bloat.

Inspiration

Inspiré du célèbre CPU-Z de CPUID, SysProbe apporte une interface graphique moderne et entièrement personnalisée, adaptée aux systèmes Windows et Linux.

SysProbe est développé en Python, avec des librairies open source. Le code source est disponible librement pour consultation, modification et contribution.

Confidentialité

Aucune donnée n'est collectée. SysProbe fonctionne entièrement en local sur votre machine. Aucune connexion réseau, aucun télémétrie.

Introduction

EBWEBS SysProbe est un logiciel d'information et de monitoring système développé par EBWEBS. Il permet de visualiser en temps réel les données clés de votre ordinateur : processeur, mémoire vive, stockage, interfaces réseau et informations système.

Inspiré de CPU-Z, SysProbe propose une interface graphique moderne 100% sur-mesure, sans barre de titre Windows native, avec des boutons de fenêtre intégrés au design EBWEBS.

ℹ

SysProbe ne collecte aucune donnée. Tout fonctionne en local sur votre machine.

Installation

Windows

  1. Télécharger et installer Python 3.8+ depuis python.org
  2. Cocher "Add Python to PATH" pendant l'installation
  3. Ouvrir un terminal (cmd ou PowerShell) et exécuter :
pip install psutil PyQt5
  1. Lancer l'application :
python ebwebs_sysprobe.py
ℹ

Sous Windows, tu peux aussi double-cliquer sur lancer_ebwebs.bat — il installe automatiquement les dépendances et lance l'appli.

Linux (Ubuntu / Debian)

# Installer les dépendances système sudo apt update sudo apt install python3-pip python3-pyqt5 # Installer psutil pip3 install psutil --break-system-packages # Lancer python3 ebwebs_sysprobe.py

Linux (Fedora / RHEL)

sudo dnf install python3-pip python3-qt5 pip3 install psutil python3 ebwebs_sysprobe.py

Linux (Arch)

sudo pacman -S python-pyqt5 python-psutil python ebwebs_sysprobe.py

Interface générale

SysProbe utilise une fenêtre entièrement personnalisée, sans la barre de titre standard de Windows ou Linux.

Barre de titre EBWEBS

  • Logo EBWEBS — à gauche, affiche le nom et la version
  • Point vert clignotant — indique que le monitoring est actif
  • Bouton Réduire — minimise la fenêtre dans la barre des tâches
  • Bouton Fullscreen — passe en plein écran / revient à la taille normale
  • Bouton Fermer — ferme l'application proprement

Déplacer la fenêtre

Cliquer et maintenir le clic gauche sur la barre de titre, puis glisser. Un double-clic sur la barre de titre passe en plein écran.

Redimensionner

Approcher le curseur du bord ou du coin de la fenêtre — le curseur change et tu peux tirer pour redimensionner dans toutes les directions.

Barre de statut

En bas de la fenêtre, la barre de statut affiche en permanence : charge CPU globale, utilisation RAM et l'heure actuelle, mis à jour chaque seconde.

Onglet Processeur

PROCESSEUR
  • Nom complet du processeur (lecture du registre Windows ou /proc/cpuinfo Linux)
  • Architecture (x86_64, ARM, etc.)
  • Nombre de cœurs physiques et de threads logiques
  • Fréquence actuelle en MHz
  • Fréquence maximale en MHz
CHARGE GLOBALE CPU
  • Pourcentage d'utilisation global du processeur, mis à jour en temps réel
  • Barre de progression colorée (bleu → rouge selon la charge)
CHARGE PAR CŒUR
  • Une carte par cœur logique (C0, C1, C2…)
  • Fréquence et pourcentage individuel de chaque cœur
  • Barre de charge miniature par cœur

Onglet Mémoire

Affiche l'état de la mémoire vive physique et de la mémoire virtuelle (swap), mis à jour toutes les 2 secondes.

MÉMOIRE RAM PHYSIQUE
  • Barre visuelle de remplissage (utilisé vs total)
  • Capacité totale installée en GB
  • Mémoire utilisée en GB
  • Mémoire disponible en GB
  • Pourcentage d'utilisation
SWAP / MÉMOIRE VIRTUELLE
  • Taille totale du SWAP
  • Espace SWAP utilisé
  • Pourcentage d'utilisation
⚠

Un taux de SWAP élevé (>50%) indique un manque de RAM physique — les performances peuvent être dégradées.

Onglet Disques

Liste tous les volumes et partitions montés sur le système.

PAR DISQUE / PARTITION
  • Identifiant du périphérique (ex: /dev/sda1, C:\)
  • Point de montage
  • Système de fichiers (NTFS, ext4, FAT32…)
  • Espace total, utilisé et libre en GB
  • Barre de remplissage colorée : vert <70%, orange <90%, rouge ≥90%

Onglet Réseau

Affiche toutes les interfaces réseau détectées sur la machine.

PAR INTERFACE
  • Nom de l'interface (ex: eth0, Wi-Fi, lo)
  • Statut : En ligne (vert) ou Hors ligne (rouge)
  • Adresse IPv4
  • Adresse MAC
  • Vitesse de liaison en Mb/s

Onglet Système

SYSTÈME D'EXPLOITATION
  • Nom et version de l'OS
  • Build / numéro de version complet
  • Architecture (64-bit / 32-bit / ARM)
  • Nom d'hôte de la machine
  • Utilisateur connecté
  • Date et heure du dernier démarrage
UPTIME
  • Temps écoulé depuis le dernier démarrage, format HH:MM:SS
  • Mis à jour chaque seconde
TEMPÉRATURES (si dispo)
  • Lecture des capteurs thermiques via psutil (Linux / Windows avec drivers)
  • Couleur verte <75% du seuil, orange <90%, rouge ≥90%

FAQ

L'appli ne se lance pas sous Windows

Vérifier que Python est bien dans le PATH : ouvrir cmd et taper python --version. Si rien n'apparaît, réinstaller Python en cochant "Add to PATH".

Les températures ne s'affichent pas

Sous Linux, installer lm-sensors : sudo apt install lm-sensors && sudo sensors-detect. Sous Windows, les capteurs nécessitent des pilotes spécifiques à la carte mère.

La fenêtre ne se déplace pas

Cliquer et maintenir sur la barre de titre EBWEBS (pas sur les boutons). Si la fenêtre est en plein écran, double-cliquer d'abord pour revenir en mode fenêtré.

PyQt5 ne s'installe pas

Essayer : pip install PyQt5 --break-system-packages. Sur Linux Debian/Ubuntu, préférer sudo apt install python3-pyqt5.

La RAM affichée ne correspond pas à ce que j'ai

SysProbe affiche la RAM disponible pour l'OS, pas la RAM physique totale. Une partie peut être réservée par le BIOS ou la carte graphique intégrée.
